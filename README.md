# Rolling <img width="30" alt="ICON" src="https://github.com/user-attachments/assets/f7c41b0b-9d8b-48f8-8a6a-d1b5cf3fcc8d" />

✉️ **"누구나 손쉽게, 온라인에서 함께 만드는 롤링 페이퍼"** <br>

친구, 가족, 동료들과 함께 한 장의 편지를 완성해보세요.<br>
익명 또는 닉네임 모드, 다양한 템플릿 제공, 간편한 공유 기능을 통해 특별한 순간을 간직할 수 있습니다.

- **프로젝트명**: Rolling  
- **팀명**: 14-8팀 😄

- **진행 방식**: JIRA를 활용해 작업을 효율적으로 분배하고, 우선순위를 명확히 하여 프로젝트 일정을 체계적으로 관리했습니다.

## ✨ 주요 기능

- 😊 **간편한 참여** – 링크 하나로 누구나 쉽게 작성 가능  
- 🎨 **커스텀 디자인** – 배경, 폰트, 테마를 자유롭게 선택 가능  
- 🔗 **공유 기능** – 완성된 롤링 페이퍼를 카카오톡 또는 링크 복사로 공유  
- 💬 **실시간 미리보기** – 다른 사용자의 작성 내용을 실시간으로 확인 가능

### 📌 사용 이유

| 기술/도구 | 사용 이유 |
|-----------|-----------|
| **React** | 컴포넌트 기반 구조로 UI를 효율적으로 관리할 수 있고, 상태 변화에 따른 UI 업데이트가 직관적이기 때문에 채택했습니다. |
| **Styled Components** | CSS-in-JS 방식으로 컴포넌트별 스타일을 캡슐화하여 유지보수가 용이하고, 조건부 스타일링이 편리해 선택했습니다. |
| **CSS3** | 기본적인 레이아웃, 애니메이션, 반응형 처리를 위한 필수 스타일링 도구로 사용했습니다. |
| **HTML5** | 시맨틱한 구조와 접근성을 고려한 마크업을 위해 사용했습니다. |
| **JavaScript** | 웹 애플리케이션의 로직 구현과 API 통신, 사용자 이벤트 처리를 위해 사용했습니다. |
| **Vite** | 빠른 개발 환경 세팅과 HMR(Hot Module Replacement) 성능을 위해 Vite를 빌드 도구로 선택했습니다. |
| **GitHub** | 협업 및 버전 관리를 위한 기본 플랫폼으로 사용했습니다. |
| **Jira** | 이슈 관리와 작업 분담, 일정 관리를 체계적으로 하기 위해 사용했습니다. |
